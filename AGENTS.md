# AGENTS.md

## 프로젝트 작업 규칙

- 구현 변경을 하기 전에 항상 다음 memory-bank 문서를 조회한다.
  - `memory-bank/architecture.md`
  - `memory-bank/implementation-plan.md`
  - `memory-bank/progress.md`
- 프로젝트 방향, 아키텍처, 구현 계획, 진행 상황이 바뀌면 관련 memory-bank 문서를 함께 수정한다.
- 의미 있는 작업을 마친 뒤에는 다음 세션이 이어서 작업할 수 있도록 `memory-bank/progress.md`를 갱신한다.
- 실제 구현 결정이 문서화된 계획과 달라지면, 같은 변경 안에서 관련 memory-bank 문서를 갱신한다.
- 문서 작성, 작업 보고, 사용자와의 커뮤니케이션은 한국어로 진행한다.
- 코드, API 이름, 파일명, 명령어, 환경 변수, 외부 서비스의 고유 명칭은 필요한 경우 원문을 유지할 수 있다.
- 문서와 코드가 서로 맞지 않으면 둘 다 확인한 뒤 현재 구현 의도에 맞게 조정한다.
- 문서화된 아키텍처와 계획을 존중하면서 작고 검토 가능한 변경을 선호한다.
- 사용자가 만든 변경사항을 덮어쓰지 않는다. 현재 파일과 memory-bank 내용이 충돌하면, 수정 전에 양쪽을 확인하고 조정한다.

## 프로젝트 목표

방문자가 사진을 첨부하거나 직접 그림을 그려 방명록을 남기고, 공유 보드에서 포스트잇처럼 확인하며, 각 방명록에 달리는 댓글이 새로고침 없이 실시간으로 반영되는 전자 방명록 웹앱을 만든다.
