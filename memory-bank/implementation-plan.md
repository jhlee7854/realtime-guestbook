# 구현 계획

## 목표

React 또는 Next.js와 Supabase Realtime, Supabase Storage를 사용해 배포 가능한 실시간 전자 방명록 웹앱을 만든다.

## 1단계: 프로젝트 설정

- 프론트엔드 프로젝트를 초기화한다.
- 라우팅 방식과 스타일링 방식을 결정한다.
- Supabase 클라이언트 의존성을 추가한다.
- Supabase URL과 anon key를 위한 환경 변수 예시를 만든다.
- 컴포넌트, 훅, 데이터 접근 함수, 타입을 위한 기본 파일 구조를 만든다.

## 2단계: Supabase 스키마와 Storage

- 데이터베이스 테이블을 만든다.
  - `guestbook_posts`
  - `comments`
- 인덱스를 추가한다.
  - `guestbook_posts.created_at`
  - `comments.post_id`
  - `comments.created_at`
- `guestbook-images` Storage 버킷을 설정한다.
- 공개 또는 익명 방명록 사용을 위한 Row Level Security 정책을 정의한다.
- 필요한 SQL과 Supabase 대시보드 설정 절차를 문서화한다.

## 3단계: 방명록 작성 화면

- 작성자 이름과 메시지 입력 필드를 만든다.
- 이미지 입력 방식 전환 UI를 만든다.
  - 사진 업로드 모드
  - 캔버스 그림 모드
- 사진 미리보기를 구현한다.
- 드로잉 캔버스를 구현한다.
  - 펜 도구
  - 색상 선택
  - 브러시 굵기 선택
  - 지우개
  - 전체 지우기
- 캔버스 그림을 PNG Blob으로 변환한다.
- 선택된 이미지를 Supabase Storage에 업로드한다.
- `guestbook_posts` 레코드를 생성한다.
- 로딩, 유효성 검사, 오류 상태를 표시한다.

## 4단계: 포스트잇 보드

- 기존 방명록을 생성 시간 기준으로 조회한다.
- 작성자, 메시지, 썸네일, 댓글 수, 작성 시간이 포함된 포스트잇을 렌더링한다.
- 포스트잇 시각 표현을 적용한다.
  - 연한 포스트잇 색상
  - 약간의 회전
  - 부드러운 그림자
- Supabase Realtime으로 새 방명록 insert 이벤트를 구독한다.
- 새 방명록이 들어오면 새로고침 없이 보드를 갱신한다.

## 5단계: 포스트잇 상세 보기와 댓글

- 선택한 포스트잇을 모달 또는 사이드 패널로 연다.
- 원본 이미지, 작성자 이름, 메시지, 댓글을 표시한다.
- 선택한 포스트잇의 기존 댓글을 조회한다.
- 댓글 작성자 이름과 댓글 내용 입력 폼을 만든다.
- 댓글을 Supabase에 저장한다.
- 선택한 포스트잇의 새 댓글 insert 이벤트를 구독한다.
- 새 댓글이 들어오면 새로고침 없이 댓글 목록을 갱신한다.
- 모달을 닫거나 다른 포스트잇으로 전환할 때 실시간 구독을 정리한다.

## 6단계: 유효성 검사와 UX 다듬기

- 폼 유효성 검사를 추가한다.
  - 작성자 이름 필수
  - 메시지 필수
  - 이미지 필수
  - 댓글 내용 필수
- 빈 상태를 추가한다.
- 로딩 상태를 추가한다.
- 오류 복구 경로를 추가한다.
- 모바일과 데스크톱 반응형 동작을 확인한다.
- 모달 키보드 조작과 포커스 처리를 확인한다.

## 7단계: 검증

- lint, 타입 검사, 빌드를 실행한다.
- 실시간 동작을 수동 검증한다.
  - 두 개의 브라우저 창에서 `/board`를 연다.
  - 한 창에서 새 방명록을 등록한다.
  - 다른 창의 보드가 새로고침 없이 갱신되는지 확인한다.
  - 두 창에서 같은 포스트잇을 연다.
  - 각 창에서 댓글을 추가한다.
  - 양쪽 댓글 목록이 새로고침 없이 갱신되는지 확인한다.
- 사진 업로드와 캔버스 그림 업로드가 모두 동작하는지 확인한다.
- Supabase Storage URL이 UI에서 정상 렌더링되는지 확인한다.

## 8단계: 문서화

- 로컬 실행 방법을 문서화한다.
- 필요한 환경 변수를 문서화한다.
- Supabase 테이블 SQL과 Storage 설정을 문서화한다.
- 실시간 동작 검증 절차를 문서화한다.
- 의미 있는 마일스톤마다 `memory-bank/progress.md`를 갱신한다.


## 구현 현황

- 1단계 프로젝트 설정은 Next.js App Router와 전역 CSS 기준으로 1차 완료되었다. 단, 현재 환경의 npm registry 403 제한 때문에 의존성 설치와 lockfile 생성은 다음 작업으로 남아 있다.
- 2단계 Supabase 스키마와 Storage 설정은 `supabase/schema.sql`과 `README.md`에 1차 문서화되었다.
- 3단계 방명록 작성 화면은 이름/메시지 입력, 사진 업로드, 캔버스 그리기, Storage 업로드, `guestbook_posts` 생성 흐름까지 1차 구현되었다.
- 4단계 포스트잇 보드는 기존 데이터 조회, 댓글 수 집계, 포스트잇 렌더링, 방명록 insert 실시간 구독까지 1차 구현되었다.
- 5단계 상세 보기와 댓글은 모달, 기존 댓글 조회, 댓글 작성, 댓글 insert 실시간 구독까지 1차 구현되었다.
- 6~7단계는 빌드/실환경 검증 후 보완이 필요하다.
